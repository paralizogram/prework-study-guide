HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element is iportant for Search Engine Optimization (SEO)
  • Child elements are nested inside Parent elements
  • HTML provides the foundational structure of a page
  • Elements signal the type of content on a webpage, such as Text, Image, Header, and List Elements
  • Elements are also used to divide and describe content, such as meta-related, content division and semantic elements

CSS

  • Inline CSS is applied to specific HTML tags
  • Internal CSS style sheets include CSS rules in head section
  • External stlye sheets are kept in separate file and linked to
  • Linked external style sheets are examples of separation of concern and Do Not Repeat Yourself (DRY) principles
  • asteriks (*) are special wild card symbols used to apply a rule to all visible elements
  • CSS box model - consists of margins outside elements, element borders, padding and content
  • Class attributes allow sharing a CSS rule to elements designated within HTML to that class
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • ls: lists files & folders in current directory
  • cd ____: changes directory
  • cd ..: navigates backwards, one level at a time
  • mkdir: make new directory
  • touch: creates file in directory
  • open (Mac) / start (Win) - opens file

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions do not execute automatically, but must be called in order to do so.
  • JavaScript executed top to bottom, so order matters.
